# AI-Assisted Development Under Schedule Pressure (Pilot Study)

Reproducibility package for:

**AI-Assisted Development Under Schedule Pressure: A Controlled Pilot Study of Micro-Task Productivity and Security Trade-offs**  
Mohammad Tanhaei and Roohallah Alizadehsani, *The Journal of Systems and Software*

## Scope

The paper reports a controlled 2 x 2 pilot experiment with 48 professional Python developers. Tool support (manual versus GitHub Copilot/GPT-4) was crossed with schedule condition (6.0-hour nominal schedule versus a 3.5-hour compressed hard cap) on a bounded hospital bed-management task.

The classical Putnam equation and Brooks' Law are used only as project-scale context. The experiment does not test, revise, or overturn either project-scale model.

## Data provenance

`data/dataset_48.csv` is a **synthetic participant-level reconstruction** that matches the manuscript's reported effort and composite-quality summaries. It is not the original participant-level dataset and must not be described as raw experimental data.

Outcomes for which only group-level information is available are stored separately:

- `quality_components_summary.csv`: rounded component means from Table 7.
- `weight_sensitivity_summary.csv`: reported success rates from Table 8.
- `security_summary.csv` and `sast_breakdown.csv`: security counts from Tables 9-10 and Section 5.7.
- `participant_balance_summary.csv`: participant-balance values from Table 2.
- `questionnaire_summary.csv`: questionnaire means reported in the appendix.

This separation prevents aggregate values from being mistaken for participant-level observations.

## What is reproducible

| Manuscript result | Public-data status |
| --- | --- |
| Table 4 group means, SDs, t-based CIs, and success rates | Recomputed from `dataset_48.csv` |
| Tables 5-6 two-way ANOVA and partial eta-squared | Recomputed from `dataset_48.csv` |
| Table 7 rounded quality-component means | Composite consistency checked from aggregate means |
| Table 8 weight sensitivity | Reported aggregate table only; participant-level components are unavailable |
| Section 5.3 mixed-effects models | Not independently reproducible; `PriorAI` and `Stratum` are unavailable |
| Section 5.5 alpha, bootstrap intervals, Welch tests, and mu | Recomputed from `dataset_48.csv` |
| Section 5.7 Wilson/Newcombe intervals and Fisher tests | Recomputed from aggregate counts; the SAST scans themselves are not reproducible without submissions and raw reports |
| Participant balance and questionnaire outcomes | Reported aggregate summaries only |

`manuscript_validation.csv`, generated by the analysis, records this mapping and identifies any manuscript-side numerical mismatch.

## Run the complete analysis

Python 3.9 or newer is required.

```bash
python -m pip install -r requirements.txt
python scripts/data.py
python scripts/analysis.py
python -m unittest discover -s tests -v
```

The Python analysis is the primary, fully tested pipeline. `scripts/analysis.R` remains as a base-R mirror of the group-summary and alpha calculations:

```bash
Rscript scripts/analysis.R
```

The seeded alpha run gives a group-level estimate near 0.206 and a bootstrap interval near `[-0.029, 0.453]`. The manuscript reports this approximately as `0.21 [-0.03, 0.46]`; the inference is unchanged because the interval crosses zero. The participant-level estimate is near 0.228 with a lower endpoint effectively at zero.

## Statistical consistency note

The standard uncorrected Wilson interval for 6/24 manual submissions is 12.0%-44.9%, and the standard Newcombe hybrid-score interval for the AI-minus-manual difference is 5.5%-54.9%. These do not equal the manuscript's current 11.2%-46.9% and 7.1%-54.9% values. The analysis uses the standard formulas and records both discrepancies in `manuscript_validation.csv`; it does not hard-code the manuscript values.

The synthetic G4 cell also contains `E_session` values above the 3.5-hour wall-clock cap (maximum 4.650) because it was constructed to reproduce the reported G4 mean 3.42 and SD 0.45. Those summary values cannot both arise from 12 observations bounded above by 3.5. The manuscript can retain them only if `E_session` is explicitly defined as a sum that may exceed wall-clock duration (for example, because activity categories overlap); this needs clarification, especially because G2 is represented by the wall-clock cap itself.

## Interpretation

The exploratory alpha is a task-specific pilot descriptor, not a stable scheduling constant. Static-analysis results are security-risk indicators rather than confirmed exploitable vulnerabilities. Findings should not be extrapolated to multi-team, multi-month projects without longitudinal replication.
